# 01 — BMP280 via I²C (Fase 1)

## O que foi feito

1. **Fiação**: VCC→3V3, GND→GND, SDA→GPIO21, SCL→GPIO22 (pull-ups já no
   módulo; internos da ESP32 habilitados em paralelo).
2. **Scanner I²C**: `i2c_new_master_bus` + `i2c_master_probe` varrendo
   0x08–0x77. Sensor respondeu em **0x76** (SDO→GND).
3. **Leitura do ID na mão**: `i2c_master_transmit_receive` (escreve 0xD0,
   lê 1 byte) → **0x58** ✓ (0x60 indicaria BME280).
4. **Componente pronto** `k0i05/esp_bmp280` (^1.2.7) via IDF Component
   Manager (`idf.py add-dependency`). Escolha: dos 4 candidatos do
   registry, era o único confirmado no driver **novo** (`i2c_master`) —
   o `esp-idf-lib/bmp280` usa o legado `i2cdev`, que não coexiste com o
   novo no mesmo firmware.
5. **Leitura periódica** a cada 5 s (temperatura °C, pressão Pa→hPa).

## Conceitos da fase

- **I²C**: barramento de 2 fios (SDA/SCL) compartilhado; open-drain +
  pull-ups; endereço de 7 bits; ACK/NACK — o scanner é um teste de ACK.
- **Registradores**: conversar com o sensor = escrever o endereço do
  registrador ler bytes. ID em 0xD0; calibração em
  0x88–0xA1; ctrl_meas em 0xF4; dados em 0xF7–0xFC.
- **Bruto vs compensado**: o ADC entrega contagens; a Bosch grava 12
  coeficientes de calibração de fábrica em cada chip, e um polinômio do
  [datasheet](./fase1/BMP280_Datasheet.pdf) (§3.11.3)  converte contagens → °C/Pa. A compensação de
  pressão depende da temperatura (`t_fine`).
- **Component Manager**: `idf_component.yml` ≈ package.json;
  `managed_components/` ≈ node_modules (fora do git);
  `dependencies.lock` ≈ package-lock.json.

## Diagnóstico: módulo BMP280 defeituoso

Sintoma: `temperatura: -142.52 C | pressao: 578 hPa` (constantes),
pressão variando levemente entre leituras.

Investigação (método: primeiro a fonte do driver, depois o hardware):

1. Fonte do componente auditada: byte order da calibração (little-endian ✓),
   montagem dos 20 bits ✓, layout de bits do ctrl_meas ✓, fórmulas ✓.
2. Dump de debug no firmware: `ctrl_meas=0x6F` lido de volta do chip
   (osrs_t=3, osrs_p=3, modo normal ✓), `dig_T1=27569` (plausível ✓),
   dados crus `F7–FC = 7C xx 8x | 00 00 00` → **raw da temperatura = 0**.
   Com raw 0 e calibração real, a fórmula produz ≈ −142 °C — bate.
3. Matriz de experimentos escrevendo ctrl_meas na mão (1x/4x/16x,
   normal/forced): pressão mede em todos; temperatura **zero em todos**.
   Detalhe: em forced o bit de modo se auto-limpa após a conversão
   (0x25→0x24), semântica correta — o chip "entende" comandos.
4. Dump completo do mapa (0x80–0xFF) em dois instantes: só 0xF8/0xF9
   (pressão bruta) variam. Não existe registrador de temperatura
   alternativo. Indício extra de clone: regiões reservadas do BMP280
   legítimo (0x80–0x87, 0xA2–0xC3) cheias de dados.

**Veredito**: ADC de temperatura morto. Como o `t_fine` da temperatura
entra na compensação da pressão, a pressão absoluta também sai errada
(578 vs ~890 hPa esperados em Brasília) → módulo inutilizável.

**Ação**: substituir o módulo. O firmware está pronto e correto; a
validação final da fase fica pendente do hardware novo.

## Comandos usados

```bash
# dentro do container (via compose run):
idf.py add-dependency "k0i05/esp_bmp280^1.2.7"
make build && make flash
make run
```
